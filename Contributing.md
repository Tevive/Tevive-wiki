# Contributing to Tevive

Thank you for your interest in contributing to Tevive.

## Before Contributing

* Read the README and EULA.
* Open an issue before making major changes.
* Follow the project's coding style.

## Pull Requests

* Keep changes focused and clear.
* Test your changes before submitting.
* Provide a clear description of what was changed.

## Code Ownership

By contributing to Tevive, you agree that your contributions may be incorporated into the project and remain subject to the project's license and ownership terms.
